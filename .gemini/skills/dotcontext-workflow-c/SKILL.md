# Dotcontext Workflow — Phase C

## When to Use
Active PREVC workflow is in phase **C (Confirmation)** and you need the phase checklist independent of MCP, CLI, hooks, or Pi.

## Checklist
1. Update documentation, README, and changelog
2. Export skills or context with `sync` when adapters need refreshed assets
3. Capture deployment or handoff notes
4. Complete the workflow with `workflow-advance` when Confirmation deliverables are done

## Expected Outputs
documentation, changelog, deploy

## Orientation
Call `workflow-guide` for live next steps, relevant skills, and gate decisions. Use `dotcontext-workflow` for the full PREVC overview.
