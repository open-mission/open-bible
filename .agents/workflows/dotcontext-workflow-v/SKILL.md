# Dotcontext Workflow — Phase V

## When to Use
Active PREVC workflow is in phase **V (Validation)** and you need the phase checklist independent of MCP, CLI, hooks, or Pi.

## Checklist
1. Run the test suite and fix failures
2. Review code quality, security, and performance risks
3. Run required sensors (`workflow-manage runSensors`) and attach evidence
4. Advance only when validation gates and acceptance criteria pass

## Expected Outputs
test-report, review-comments, approval

## Orientation
Call `workflow-guide` for live next steps, relevant skills, and gate decisions. Use `dotcontext-workflow` for the full PREVC overview.
