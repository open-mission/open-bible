## Reporting a Vulnerability

We take the security of Open Bible seriously. If you believe you have found
a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via:

- **GitHub Security Advisory**: Use the "Report a vulnerability" form at
  https://github.com/open-mission/open-bible/security/advisories/new
- **Email**: Send details to the maintainers (refer to the commit history
  for contact information)

You should receive a response within 48 hours. If for some reason you do not,
please follow up to ensure we received your original message.

Please include the following information in your report:

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## Preferred Languages

We prefer all communications to be in English or Portuguese.

## Disclosure Timeline

We follow a 90-day disclosure timeline. We ask that you give us 90 days to
address the issue before public disclosure.

## Attribution

We appreciate and credit security researchers who help keep our users safe.
If you report a vulnerability, we will acknowledge your contribution in our
release notes (unless you prefer to remain anonymous).
